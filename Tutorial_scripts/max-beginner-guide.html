Open Max and Create Your First Patcher

Open Max/MSP
Click File ‚Üí New Patcher
A blank gray window will appear. This is your "patcher" - think of it as a blank canvas where you'll build your guitar amp.

Create the RNBO Container

Press the N key on your keyboard

A small box will appear with a cursor blinking inside.

Type: rnbo~
Press Enter
rnbo~
The rnbo~ object is a container. Everything we build inside it can be exported to the web!
Double-click the rnbo~ object

A new window opens - this is the RNBO editor. This is where we'll build our amp.

Keep both windows open! The main patcher and the RNBO editor window. We'll work inside the RNBO editor.
3

Create Audio Input

Now we're working INSIDE the RNBO editor window.

Press N key
Type: adc~ 1
Press Enter
adc~ 1 AUDIO
adc~ = "Analog to Digital Converter"
This captures audio from your interface. The "1" means channel 1 (left input).
4

Add Input Gain Control

Let's add volume control for the input signal.

Press N below your adc~ object
Type: *~ 1
Press Enter
adc~ 1
‚Üì
*~ 1
*~ = "multiply audio signal"
This multiplies the audio by a number. 1 = no change, 2 = double volume, 0.5 = half volume.

Connect Them:

Click the small circle at the BOTTOM of adc~ 1 (the outlet)
Drag down to the LEFT circle at the TOP of *~ 1 (the inlet)
Release the mouse
You'll see a striped line connecting them. Striped lines = audio signals!
5

Make Input Gain Controllable

Now let's make the gain adjustable from our web interface!

Press N to the RIGHT of your *~ object
Type: param inputGain @min 0 @max 2 @initial 1
Press Enter

@min 0 @max 2 @initial 1
Connect the param output to the RIGHT inlet of *~
Left inlet = audio signal, Right inlet = control value
Click the outlet of param inputGain

What this means:
‚Ä¢ @min 0 = minimum value is 0 (silent)
‚Ä¢ @max 2 = maximum value is 2 (double volume)
‚Ä¢ @initial 1 = starts at 1 (normal volume)

First, add the drive control:

Press N below your *~ object
Type: *~ 5

Program the distortion:

Double-click the codebox~ object

What's happening:
‚Ä¢ We multiply the signal by 1-20 (the drive amount)
‚Ä¢ The tanh() function "clips" the signal smoothly

Add output compensation:

Press N below codebox~
Type: *~ 0.3
Press Enter
Connect codebox~ output to this *~ 0.3 input
This reduces the volume back down after distortion so it doesn't blow your speakers!

Add Tone Control (Treble Cut)

Let's add a simple tone knob to control brightness.

Press N below your last *~ object
Type: biquad~ @mode lowpass @Q 0.707
Press Enter
Press N to the right
Type: param tone @min 500 @max 8000 @initial 5000
Press Enter
*~ 0.3
(distortion output from previous step)

‚Üê

Connect the audio to the LEFT inlet of biquad~
Connect the param tone to the frequency inlet (usually inlet 2 - second from left)
biquad~ is a filter. In lowpass mode:
‚Ä¢ Low frequencies pass through
‚Ä¢ High frequencies get cut
‚Ä¢ Turning tone down = darker sound
‚Ä¢ Turning tone up = brighter sound
8

Add Master Volume

Press N below biquad~
Type: *~ 0.5
Press Enter
Press N to the right
Type: param master @min 0 @max 1 @initial 0.5
Press Enter
Connect biquad~ output to *~ 0.5 left inlet
Connect param master to *~ 0.5 right inlet
This controls the final output volume. We keep max at 1 to prevent clipping.

Add limiter:

Press N below master volume

Press Enter
Connect the master volume output to clip~ input
clip~ prevents the signal from going above 0.95 or below -0.95, protecting your ears and speakers!

Add audio output:

Press N below clip~
Type: dac~ 1
Press Enter

Test Your Amp in Max

Setup audio:

In Max menu: Options ‚Üí Audio Status
Set "Driver" to your Focusrite interface
Set "Input Device" to your Focusrite
Set "Output Device" to your Focusrite (or computer speakers for testing)
Close the Audio Status window

Turn on audio:

Look at the bottom-right corner of Max window
Click the speaker icon to turn audio ON
The speaker icon should highlight when audio is on. If you see a red "x", check your audio settings!

Lock the patch and test:

Press Cmd+E (Mac) or Ctrl+E (Windows)

The patch is now "locked" - you can interact with it but can't edit it.

Plug in your guitar
Play some notes - you should hear your guitar processed through the amp!
If you don't hear anything:
‚Ä¢ Check if audio is on (speaker icon)
‚Ä¢ Make sure your guitar is plugged into the right input
‚Ä¢ Try increasing the inputGain and master volume
‚Ä¢ Check that your interface is receiving signal (input LED blinking)

Adjust parameters:

Unlock the patch: Press Cmd+E / Ctrl+E again
Add sliders to control parameters (optional for testing)
Or just manually change the numbers in the param objects

Common Problems & Solutions

Problem: No sound in Max

Solutions:
‚Ä¢ Turn on audio (speaker icon bottom-right)
‚Ä¢ Check Options ‚Üí Audio Status
‚Ä¢ Make sure your interface is selected
‚Ä¢ Lock the patch (Cmd/Ctrl+E) before testing
‚Ä¢ Increase inputGain and master parameters

Problem: Can't connect objects

Solutions:
‚Ä¢ Make sure patch is UNLOCKED (Cmd/Ctrl+E)
‚Ä¢ Click the outlet (bottom circle) first
‚Ä¢ Drag to the inlet (top circle)
‚Ä¢ Audio connections should be striped lines

Problem: Export fails

Solutions:
‚Ä¢ Save your patch first
‚Ä¢ Make sure you're using RNBO-compatible objects only
‚Ä¢ Check for any error messages in Max window
‚Ä¢ Try exporting to a different location

Problem: Distortion sounds bad

Solutions:
‚Ä¢ Lower the drive parameter
‚Ä¢ Reduce inputGain
‚Ä¢ Try different codebox algorithms
‚Ä¢ Add more filtering after distortion

Problem: React app can't find patch.export.json

Solutions:
‚Ä¢ Verify file is in public/ folder
‚Ä¢ File name must be exactly "patch.export.json"
‚Ä¢ Restart React development server (npm start)
‚Ä¢ Check browser console for specific error
